# Исследование Муравьиного алгоритма (Ant Colony Optimization)

## Обзор

Этот проект представляет собой исследовательский проект для изучения и оценки алгоритма **Муравьиной колонии (Ant Colony Optimization — ACO)** на задачах поиска кратчайшего пути на связном взвешанном графе.

Основная цель — предоставить **воспроизводимую и расширяемую среду** для сравнения ACO с классическими алгоритмами:

- Дейкстры (Dijkstra)
- A*
- Флойда–Уоршелла (Floyd–Warshall)

Скрипт автоматически:
- генерирует разнообразные тестовые графы,
- запускает все алгоритмы на едином наборе данных,
- собирает результаты,
- выполняет сравнительный анализ.

## Сборка (Docker)

Собрать проект с помощью Docker:

```bash
docker build --no-cache -t aco-project .
```
## Запуск проекта
Рекомендуемый способ (с монтированием папок):
```bash
docker run -it --rm \
  -v "$(pwd)/results:/runtime/results" \
  -v "$(pwd)/data:/runtime/data" \
  aco-project ./bin/aco --full-pipeline
```
Это позволит сохранять результаты напрямую на ваш компьютер.

## Доступные команды
**Показать справку**:
```bash
./bin/aco --help
```
**Полный пайплайн (рекомендуется)**:
```bash
./bin/aco --full-pipeline
```
Эта команда выполняет:
- генерацию тестов (если их ещё нет)
- запуск всех алгоритмов
- сравнительный анализ

**Запуск на одном графе**:
```bash
./bin/aco --single <graph_file>
```
**Генерация тестового набора**:
```bash
./bin/aco --generate-tests
```
**Запуск конкретного алгоритма**:
```bash
./bin/aco --run aco
./bin/aco --run dj
./bin/aco --run fw
./bin/aco --run astar
```
**Запуск всех алгоритмов**:
```bash
./bin/aco --run-all
```
**Анализ существующих результатов**:
```bash
./bin/aco --analyze
```

## Результаты

### Файлы результатов алгоритмов
```
aco_results.csv         # результаты алгоритма ACO
dijkstra_results.csv    # результаты алгоритма Дейкстры
astar_results.csv       # результаты алгоритма A*
fw_results.csv          # результаты алгоритма Флойда — Уоршелла
```

Результаты включают в себя:
- название тестового файла,
- описание графа из этого файла (число вершин и рёбер),
- время работы алгоритма на этом графе
- длина найденного кратчайшего пути,
- факт нахождения кратчайшего пути,
- количество итераций алгоритма на графе (для всех, кроме ACO, Iterations = 1),
- последовательность вершин кратчайшего пути.

**experiment_meta.json** — метаданные запуска эксперимента.

### Файлы анализа и сравнения алгоритмов
```
detailed_comparison.csv # подробная агрегированная статистика по алгоритмам и категориям тестов
category_comparison.csv # сравнение алгоритмов по категориям графов (разбиение на категории по размеру графа, плотности и виду)
per_test_comparison.csv # самый подробный файл сравнения по каждому тесту всех алгоритмов
algorithm_rankings.csv  # итоговое ранжирование алгоритмов по скорости, надёжности, точности и ошибке
```

**summary_comparison.csv** — краткая сводная таблица по алгоритмам, включающая в себя метрики:
- процента тестов, на которых был найден кратчайший путь,
- среднего время выполнения тестов,
- суммарного время выполнения всех тестов алгоритмом,
- процента тестов, на которых длина кратчайшего пути совпала с эталонной длиной алгоритма Дейкстры,
- средней относительной ошибки длины кратчайшего пути относительно Дейкстры,
- максимальная относительная ошибка.

## Примечание

Фреймворк детерминирован благодаря фиксированным зернам генератора случайных чисел (для воспроизводимости результатов).
Все алгоритмы оцениваются на одинаковых наборах графов.
Система спроектирована для удобного расширения (динамические графы, ML-управляемый ACO и т.д.).
